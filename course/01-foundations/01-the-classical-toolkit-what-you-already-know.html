01 - The Classical Toolkit (What You Already Know)

#classical #foundations #fiji

Overview

Before we dive into AI, let's acknowledge and document the classical workflow that has served the microscopy community for decades. Understanding what these tools do well — and where they break down — is essential context for appreciating why deep learning changed everything.

The Classical ImageJ Workflow

Every image analysis pipeline you've ever built probably followed this pattern:

Raw Image
    → Pre-processing (filters, background subtraction)
        → Segmentation (thresholding)
            → Post-processing (watershed, morphological operations)
                → Measurement (Analyze Particles)
                    → Results Table / ROI Manager

Step 1: Pre-Processing

Goal: Improve signal-to-noise ratio and remove artifacts.

Tool Menu Path What It Does
Gaussian Blur Process → Filters → Gaussian Blur Smooths noise, loses sharp edges
Median Filter Process → Filters → Median Removes salt-and-pepper noise, preserves edges better
Rolling Ball BG Subtraction Process → Subtract Background Removes uneven illumination
Enhance Contrast Process → Enhance Contrast Stretches histogram for better visibility

Macro example — basic pre-processing:

// Classic pre-processing pipeline
run("Subtract Background...", "rolling=50");
run("Median...", "radius=2");
run("Enhance Contrast...", "saturated=0.3");

Step 2: Segmentation (Thresholding)

Goal: Convert grayscale image to binary (object vs. background).

Common thresholding algorithms:

Method Best For Limitation
Otsu Bimodal histograms (clear foreground/background) Fails with uneven illumination
Triangle Skewed histograms Less effective on symmetric distributions
Li Minimizing cross-entropy Can be unpredictable
Manual When you know your data Not reproducible, not scalable

Macro example:

// Auto-threshold with Otsu
run("8-bit");
setAutoThreshold("Otsu dark");
run("Convert to Mask");

Step 3: Post-Processing

Goal: Clean up the binary mask — fill holes, remove debris, separate touching objects.

Tool What It Does Limitation
Fill Holes Process → Binary → Fill Holes Only fills fully enclosed holes
Open / Close Process → Binary → Open/Close Removes small objects / bridges gaps
Erode / Dilate Process → Binary → Erode/Dilate Shrinks / expands objects
Watershed Process → Binary → Watershed Attempts to split touching objects

The Watershed problem:

Watershed is the classical solution for separating touching objects. It works by finding the "ridgeline" between intensity peaks.

When it works: Fairly round, uniformly-sized objects with clear intensity gradients.

When it fails:
- Irregularly shaped cells
- Objects with uneven intensity
- Highly overlapping objects
- Objects of very different sizes

// Classic post-processing
run("Fill Holes");
run("Watershed");

Step 4: Measurement

Goal: Count objects and measure their properties.

// Set measurements and analyze
run("Set Measurements...", "area mean perimeter shape integrated redirect=None decimal=3");
run("Analyze Particles...", "size=100-Infinity circularity=0.3-1.00 show=Outlines display clear summarize add");

Analyze Particles parameters:

Parameter What It Controls
Size (min-max) Filter objects by area in pixels²
Circularity (0-1) Filter by roundness (1 = perfect circle)
Show What to display (Outlines, Masks, etc.)
Add Add detected objects to ROI Manager

A Complete Classical Macro

Here's a typical "old school" macro that most Fiji users would recognize:

// ============================================
// Classical Nuclei Counting Pipeline
// ============================================

// Pre-processing
run("Duplicate...", "title=working");
run("8-bit");
run("Subtract Background...", "rolling=50");
run("Gaussian Blur...", "sigma=2");

// Segmentation
setAutoThreshold("Otsu dark");
run("Convert to Mask");

// Post-processing
run("Fill Holes");
run("Open");
run("Watershed");

// Measurement
run("Set Measurements...", "area mean perimeter shape redirect=None decimal=3");
run("Analyze Particles...", "size=50-5000 circularity=0.5-1.00 show=Outlines display clear summarize add");

// Report
print("Total nuclei detected: " + nResults);
This macro is good enough
For simple, high-contrast, well-separated objects, this workflow is still perfectly fine. AI doesn't replace it for trivial cases — it replaces it for the cases where this fails.

What This Workflow Does Well

  • Fast and deterministic — same parameters always give same results
  • No training data required
  • Runs on any computer, no GPU needed
  • Well-documented, decades of literature
  • Fully reproducible when parameters are recorded

Where It Falls Apart

These are the exact cases that motivated the AI revolution. We'll examine each in detail in 02 - Why Classical Methods Hit a Wall.

  • Touching/overlapping objects that watershed can't separate
  • Low signal-to-noise images
  • Irregular cell shapes with no clear boundary
  • Heterogeneous staining intensity
  • Large datasets where manual threshold tuning doesn't scale
  • 3D volumes with anisotropic resolution
  • Time-lapse data with varying conditions across frames

Key Takeaway

The classical toolkit is a rule-based approach: you define explicit rules (brightness thresholds, size filters, shape criteria) and the computer follows them. AI introduces a learning-based approach: you show the computer examples, and it learns the rules itself.
