Lab 04 — BioImage Model Zoo in DeepImageJ

#lab #deepimagej #bioimage-model-zoo #hands-on

Objective

Download a pre-trained model from the BioImage Model Zoo and run it inside Fiji using DeepImageJ.

Time: ~30 minutes
Prerequisites: Fiji with DeepImageJ plugin installed

Step 1: Browse the BioImage Model Zoo

  1. Open https://bioimage.io in your browser
  2. Click Models to browse
  3. Filter by your use case — try searching for "nuclei segmentation"
  4. Find a model that matches your imaging modality
  5. Read the model card: what data was it trained on? What input does it expect?

Step 2: Install the Model in Fiji

Method A: Download and install manually
1. Download the model .zip from bioimage.io
2. In Fiji: Plugins → DeepImageJ → DeepImageJ Install Model
3. Select the downloaded .zip file
4. Wait for installation to complete

Method B: Install from within Fiji
1. Plugins → DeepImageJ → DeepImageJ Install Model
2. Browse the available models list
3. Select one and install

Step 3: Run the Model

  1. Open a test image (or use the test image provided with the model)
  2. Plugins → DeepImageJ → DeepImageJ Run
  3. Select your installed model from the dropdown
  4. Configure any pre/post-processing options
  5. Click Run
  6. Examine the output

Step 4: Post-Process the Output

DeepImageJ outputs vary by model:
- Probability map: Threshold to get a binary mask
- Label image: Each object has a unique value — convert to ROIs
- Binary mask: Ready for Analyze Particles

// If output is a probability map
selectWindow("DeepImageJ_output");
setAutoThreshold("Default");
run("Convert to Mask");
run("Analyze Particles...", "size=50-Infinity show=Outlines display clear add");

Step 5: Evaluate

Compare the DeepImageJ result against:
- Your classical pipeline result
- Manual annotation (ground truth) if available
- Your intuition from looking at the image

Takeaways

  • The BioImage Model Zoo provides ready-to-use AI models for many tasks
  • DeepImageJ makes running these models as simple as using any Fiji plugin
  • Always validate model output on your specific data before trusting it for analysis
  • If the pre-trained model doesn't work perfectly, consider fine-tuning (see 13 - ZeroCostDL4Mic - Train Models for Free)
